# Field Service Tracker — Claude Code Guide

## 1. Project
A field service management tool for tracking jobs, technicians, and client work orders.

## 2. Tech Stack

| Layer    | Stack                                              |
|----------|----------------------------------------------------|
| Backend  | FastAPI, Python 3.12, SQLAlchemy async, uv         |
| Frontend | React 18, Vite, Tailwind CSS, Radix UI             |
| Infra    | Docker Compose, PostgreSQL 16                      |

## 3. Key Commands

```bash
# Start everything
docker compose up --build

# Backend only (from backend/)
uv run uvicorn app.main:app --reload

# Run migrations
docker compose exec backend uv run alembic upgrade head

# Seed data
docker compose exec backend python -m app.seed

# Run backend tests
docker compose exec backend uv run pytest

# Frontend install + dev (from frontend/)
npm install && npm run dev
```

## 4. Pre-Commit Checklist (ALWAYS run before committing)

1. Read `COMMITS.md` — verify message format, type, scope, body length >200 chars, no AI attribution
2. Confirm only files related to the single logical change are staged
3. For backend changes: run `uv run pytest` and confirm all tests pass
4. For frontend changes: run `npm run build` and confirm no TypeScript errors

## 5. Code Conventions

- All FastAPI endpoints and SQLAlchemy queries must be `async def`
- UUIDs are generated by PostgreSQL (`gen_random_uuid()`), never by the application
- Business logic lives in `services/`, not in `routers/`
- Status transitions raise `HTTPException(400)` with descriptive messages
- TanStack Query keys: structured arrays `["resource", filters]` — use `invalidateQueries`, never manual cache edits
- CORS allows `http://localhost:5173` only

## 6. Reference Files

- `COMMITS.md` — commit message rules (read before every commit)
- `docker-compose.yml` — service definitions and env vars
- `backend/pyproject.toml` — Python deps managed by uv
